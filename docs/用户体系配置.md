# PanSou 用户体系配置指南（MySQL）

## 概述

PanSou 支持基于 MySQL 的用户体系，包括用户注册、登录、资料查询、图形验证码等功能。密码使用 bcrypt 加密存储。

**重要**：启用 `DB_ENABLED=true` 时，认证会自动开启，搜索等接口需携带 JWT，无需再单独配置 `AUTH_ENABLED`。

## 配置方式

所有配置支持通过 **.env 文件** 或 **环境变量** 设置。启动时会自动加载项目根目录的 `.env` 文件。

```bash
# 复制示例配置并编辑
cp .env.example .env
# 编辑 .env 填入你的配置
```

## 环境变量配置

### 启用用户体系

```bash
# 启用数据库（MySQL用户体系），启用后自动开启认证
export DB_ENABLED=true

# MySQL 连接配置（二选一）

# 方式1：使用完整 DSN
export DB_DSN="user:password@tcp(127.0.0.1:3306)/pansou?charset=utf8mb4&parseTime=True"

# 方式2：分项配置
export DB_HOST=127.0.0.1
export DB_PORT=3306
export DB_USER=root
export DB_PASSWORD=your_password
export DB_NAME=pansou

# JWT 配置（建议生产环境手动设置）
export AUTH_JWT_SECRET=your-random-secret-key
export AUTH_TOKEN_EXPIRY=24

# 图形验证码（可选，登录/注册时防暴力破解）
export CAPTCHA_ENABLED=true
```

### 创建数据库

首次使用前，需要先创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS pansou CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

表结构会在服务启动时自动创建，无需手动执行。

## API 接口

### 获取图形验证码

当 `CAPTCHA_ENABLED=true` 时，登录/注册前需先获取验证码。

```
GET /api/auth/captcha
```

**响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "captcha_id": "xxx",
    "captcha_image": "data:image/png;base64,iVBORw0KGgo..."
  }
}
```

前端可将 `data.captcha_image` 直接用于 `<img src="...">` 显示，`data.captcha_id` 用于登录/注册时提交。

### 用户注册

```
POST /api/auth/register
Content-Type: application/json

{
  "username": "testuser",   // 必填，3-32位字母数字下划线
  "password": "password123", // 必填，至少6位
  "email": "user@example.com", // 可选
  "captcha_id": "xxx",       // CAPTCHA_ENABLED 时必填
  "captcha_answer": "1234"   // CAPTCHA_ENABLED 时必填
}
```

**成功响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "注册成功",
    "username": "testuser",
    "id": 1
  }
}
```

### 用户登录

```
POST /api/auth/login
Content-Type: application/json

{
  "username": "testuser",
  "password": "password123",
  "captcha_id": "xxx",       // CAPTCHA_ENABLED 时必填
  "captcha_answer": "1234"   // CAPTCHA_ENABLED 时必填
}
```

**成功响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGc...",
    "expires_at": 1234567890,
    "username": "testuser"
  }
}
```

### 获取用户资料（需认证）

```
GET /api/user/profile
Authorization: Bearer <token>
```

### 验证 Token

```
POST /api/auth/verify
Authorization: Bearer <token>
```

## API 响应格式

所有接口统一使用以下格式：

**成功**：
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

**失败**：
```json
{
  "code": 400,
  "message": "错误描述"
}
```

## 兼容模式

- **DB_ENABLED=true**：使用 MySQL，支持注册、bcrypt 密码；**同时自动启用认证**，搜索等接口需携带 JWT
- **AUTH_ENABLED=true 且 DB_ENABLED=false**：使用环境变量 `AUTH_USERS`（格式：`user1:pass1,user2:pass2`），密码明文，不支持注册

**说明**：启用 MySQL 用户体系（DB_ENABLED=true）时，认证会自动开启，无需再单独配置 AUTH_ENABLED。

## Docker 部署示例

```bash
# 使用 .env 文件，推荐
cp .env.example .env
# 编辑 .env 配置 DB_ENABLED、DB_DSN 等

docker run -d --name pansou -p 8888:8888 --env-file .env ghcr.io/fish2018/pansou:latest
```

或使用 Docker Compose（从 .env 加载配置）：
```bash
cp .env.example .env
# 编辑 .env 后执行
docker-compose up -d
```

**Docker 连接宿主机 MySQL**：在 .env 中设置 `DB_HOST=host.docker.internal`（Mac/Windows）或宿主机 IP。
